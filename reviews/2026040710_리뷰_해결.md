## 3-1. API 응답 데이터를 innerHTML에 직접 넣으면 보안에 취약해요 (중요도: 높음)

### 문제

`src/main/main_list/main_list.js`의 `renderFeaturedCard()`와 `renderSmallCard()` 함수에서 서버 데이터를 **그대로 HTML에 삽입**하고 있어요. 만약 누군가 영화 제목에 `<script>` 코드를 넣으면, 다른 사용자의 브라우저에서 그 코드가 실행될 수 있어요 (XSS 공격).

### 해결 방법

**Step 1.** `src/main/main_list/main_list.js` 파일 맨 위에 안전한 텍스트 변환 함수를 추가하세요.

```js
// 수정 전 (src/main/main_list/main_list.js 1~3번째 줄)
import "./main_list.css";
import { renderHeader } from "../header/header.js";
import { getMainList } from "../../API/main_list.js";

// 수정 후 — escapeHTML 함수 추가
import "./main_list.css";
import { renderHeader } from "../header/header.js";
import { getMainList } from "../../API/main_list.js";

/**
 * HTML 특수문자를 안전한 문자로 변환해요.
 * 예: "<script>" → "&lt;script&gt;" (화면에 글자로 표시됨, 코드로 실행 안 됨)
 */
function escapeHTML(str) {
  if (str == null) return "";
  const div = document.createElement("div");
  div.textContent = String(str);
  return div.innerHTML;
}
```

**Step 2.** `renderFeaturedCard()` 함수를 수정하세요.

```js
// 수정 전 (src/main/main_list/main_list.js 32~48번째 줄)
function renderFeaturedCard(post) {
  return `
    <a href="/src/main/detail/detail.html?postId=${post.postId}" class="featured-card">
      <div class="featured-card__poster">
        ${post.imageUrl ? `<img src="${post.imageUrl}" alt="${post.title}" />` : ""}
      </div>
      <div class="featured-card__info">
        <h2 class="featured-card__title">${post.title}</h2>
        <p class="featured-card__meta">
          ${post.year ? `${post.year} · ` : ""}${GENRE_MAP[post.genre] ?? post.genre ?? ""}${post.star != null ? ` · ★ ${post.star}` : ""}
        </p>
        ${post.director?.length ? `<p class="featured-card__crew">감독: ${post.director.join(", ")}</p>` : ""}
        ${post.cast?.length ? `<p class="featured-card__crew">출연: ${post.cast.join(", ")}</p>` : ""}
      </div>
    </a>
  `;
}

// 수정 후 — escapeHTML로 감싸기
function renderFeaturedCard(post) {
  const title = escapeHTML(post.title);
  const imageUrl = escapeHTML(post.imageUrl);
  const genre = escapeHTML(GENRE_MAP[post.genre] ?? post.genre ?? "");
  const year = escapeHTML(post.year);
  const star = post.star != null ? escapeHTML(post.star) : null;
  const director = post.director?.map(escapeHTML) ?? [];
  const cast = post.cast?.map(escapeHTML) ?? [];

  return `
    <a href="/src/main/detail/detail.html?postId=${escapeHTML(post.postId)}" class="featured-card">
      <div class="featured-card__poster">
        ${post.imageUrl ? `<img src="${imageUrl}" alt="${title}" />` : ""}
      </div>
      <div class="featured-card__info">
        <h2 class="featured-card__title">${title}</h2>
        <p class="featured-card__meta">
          ${post.year ? `${year} · ` : ""}${genre}${star != null ? ` · ★ ${star}` : ""}
        </p>
        ${director.length ? `<p class="featured-card__crew">감독: ${director.join(", ")}</p>` : ""}
        ${cast.length ? `<p class="featured-card__crew">출연: ${cast.join(", ")}</p>` : ""}
      </div>
    </a>
  `;
}
```

**Step 3.** `renderSmallCard()` 함수도 같은 방식으로 수정하세요.

```js
// 수정 전 (src/main/main_list/main_list.js 50~62번째 줄)
```

## 3-2. accessToken이 콘솔에 출력되고 있어요 (중요도: 높음)

### 문제

`src/API/main_list.js`의 3~4번째 줄에서 API 요청 전에 accessToken을 콘솔에 출력하고 있어요. 이 토큰은 사용자의 로그인 인증 정보라서 노출되면 안 돼요.

### 해결 방법

**Step 1.** `src/API/main_list.js`에서 디버그용 console.log를 모두 제거하세요.

```js
// 수정 전 (src/API/main_list.js 전체)
export const getMainList = async () => {
  try {
    console.log("Fetching main list from API...");
    console.log(`accessToken: ${localStorage.getItem("accessToken")}`);
    const response = await fetch(
      `${import.meta.env.VITE_API_BASE_URL}/movies/home`,
      {
        method: "GET",
        headers: {
          "Content-Type": "application/json",
          Authorization: `Bearer ${localStorage.getItem("accessToken")}`,
        },
      },
    );
    if (!response.ok) {
      throw new Error("Failed to fetch main list");
    }
    const json = await response.json();
    console.log("Main list response:", json);
    return json.data;
  } catch (error) {
    console.error("Error fetching main list:", error);
    return null;
  }
};

// 수정 후 — 디버그 로그 제거
export const getMainList = async () => {
  try {
    const response = await fetch(
      `${import.meta.env.VITE_API_BASE_URL}/movies/home`,
      {
        method: "GET",
        headers: {
          "Content-Type": "application/json",
          Authorization: `Bearer ${localStorage.getItem("accessToken")}`,
        },
      },
    );

      throw new Error("Failed to fetch main list");
    }
    const json = await response.json();
    return json.data;
  } catch (error) {
    console.error("Error fetching main list:", error);
    return null;
  }
};
```

> **쉽게 말하면:** API 디버깅이 필요하면 브라우저 **개발자 도구(F12) → Network 탭**을 사용하세요! 거기서 요청/응답을 안전하게 확인할 수 있어요. 코드에 console.log를 남기면 누구나 토큰을 볼 수 있어요.

## 3-3. `.env` 파일이 `.gitignore`에 없어요 — 긴급! (중요도: 높음)

### 문제

`.env` 파일이 존재하는데 `.gitignore`에 포함되어 있지 않아요. Git에 커밋되면 GitHub에 공개돼요.

### 해결 방법

**Step 1.** `.gitignore` 파일을 수정하세요.

```gitignore
# 수정 전 (.gitignore)
node_modules/
dist/
build/

# 수정 후 (.gitignore)
# 의존성
node_modules/

# 빌드 결과물
dist/
build/

# 환경 변수 (API 키 등 민감 정보!)
.env
.env.local
.env.production

# OS 파일
.DS_Store
Thumbs.db

# 에디터 설정
.vscode/
.idea/
```

**Step 2.** 팀원용 예시 파일을 만드세요.

```bash
# .env.example (새로 생성 — 이 파일은 Git에 올려도 됨)
VITE_API_BASE_URL=https://여기에-API-주소를-넣으세요
```

**Step 3.** 만약 `.env`가 이미 Git에 커밋되었다면, 아래 명령으로 추적을 해제하세요.

```bash
# 터미널에서 실행
git rm --cached .env
git commit -m "Fix: .env 파일 Git 추적 해제"
```

> **쉽게 말하면:** `.env`는 비밀번호가 적힌 메모장이에요. `.gitignore`에 추가하는 건 "이 메모장은 절대 복사해서 남에게 주지 마!"라고 표시하는 거예요.

## 3-4. header.js에서 대상 요소가 없을 때 에러가 나요 (중요도: 중간)

### 문제

`src/main/header/header.js`의 31번째 줄에서 `querySelector()`의 결과를 확인하지 않아서, 대상 요소가 없으면 에러가 발생해요.

### 해결 방법

**Step 1.** `src/main/header/header.js`의 `renderHeader` 함수를 수정하세요.

```js
// 수정 전 (src/main/header/header.js 31~32번째 줄)
const target = document.querySelector(targetSelector);
target.insertAdjacentHTML("afterbegin", html);

// 수정 후
const target = document.querySelector(targetSelector);
if (!target) {
  console.error(`[Header] 대상 요소를 찾을 수 없어요: "${targetSelector}"`);
  return;
}
target.insertAdjacentHTML("afterbegin", html);
```

> **쉽게 말하면:** 택배 기사님이 "101호에 배달해줘"라고 받았는데 101호가 없으면? 무작정 멈추지 말고 "101호가 없어요!"라고 알려줘야 다음에 고칠 수 있어요.

## 3-5. Vite에서 `public/` 폴더 안 파일 경로가 잘못되어 있어요 (중요도: 중간)

### 문제

`src/main/header/header.js`의 8, 11번째 줄과 `src/main/header/header.html`의 23, 28번째 줄에서 `/public/` 접두사를 사용하고 있어요. Vite에서 `public/` 안의 파일은 루트(`/`)에서 바로 접근해야 해요.

### 해결 방법

**Step 1.** `src/main/header/header.js`를 수정하세요.

```js
// 수정 전 (src/main/header/header.js 8번째 줄)
<img src="/public/Bui1dBox.png" alt="빌드업 텍스트 로고" class="header-text-logo" />

// 수정 후
<img src="/Bui1dBox.png" alt="빌드업 텍스트 로고" class="header-text-logo" />
```

```js
// 수정 전 (src/main/header/header.js 11번째 줄)
<img src="/public/Search_icon.svg" alt="검색" class="search-icon" />

// 수정 후
<img src="/Search_icon.svg" alt="검색" class="search-icon" />
```

**Step 2.** `src/main/header/header.html`도 같은 방식으로 수정하세요.

```html
<!-- 수정 전 (header.html 23번째 줄) -->
<img src="/public/Bui1dBox.png" alt="빌드업 텍스트 로고" class="header-text-logo" />
<!-- 수정 후 -->
<img src="/Bui1dBox.png" alt="빌드업 텍스트 로고" class="header-text-logo" />

<!-- 수정 전 (header.html 28번째 줄) -->
<img src="/public/Search_icon.svg" alt="검색" class="search-icon" />
<!-- 수정 후 -->
<img src="/Search_icon.svg" alt="검색" class="search-icon" />
```

**왜 이렇게 되는지 알아볼까요?**

```
프로젝트 폴더 구조:
Bui1d-UP/
├── public/
│   └── Bui1dBox.png     ← 실제 파일 위치
└── src/
    └── ...

Vite가 서빙하는 URL:
http://localhost:5173/Bui1dBox.png     ✅ 올바른 경로
http://localhost:5173/public/Bui1dBox.png  ❌ 잘못된 경로 (public/public/을 찾음)
```

> **쉽게 말하면:** `public/` 폴더는 "택배 보관함"이에요. 보관함에 넣은 물건은 "보관함 번호(파일명)"만 말하면 꺼낼 수 있어요. "/보관함/보관함번호"라고 하면 찾을 수 없어요!

## 3-6. header.html과 header.js의 HTML이 서로 달라요 (중요도: 중간)

### 문제

헤더의 HTML이 `header.html`과 `header.js` 두 곳에 존재하고, 이미 링크 경로가 서로 달라졌어요. 이렇게 같은 코드가 두 곳에 있으면 하나만 수정하고 다른 하나를 잊어버리기 쉬워요.

### 해결 방법

**Step 1.** `src/main/header/header.html`을 수정해서, `header.js`의 `renderHeader()`를 사용하게 만드세요.

```html
<!-- 수정 전 (src/main/header/header.html 전체) -->
<!doctype html>
<html lang="ko-KR">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>헤더</title>
    <link rel="shortcut icon" href="/favicon/likelion.ico" type="image/x-icon" />
    <link rel="icon" href="/favicon/likelion.svg" type="image/svg+xml" />
    <link rel="preload" as="font" href="/font/suit/SUIT-Variable.woff2" />
    <link rel="stylesheet" href="/font/suit/SUIT-Variable.css" />
    <link rel="stylesheet" href="/src/main/header/header.css" />
  </head>
  <body>
    <header class="site-header">
      <!-- ... 많은 HTML ... -->
    </header>
  </body>
</html>

<!-- 수정 후 — header.js를 import해서 사용 -->
<!doctype html>
<html lang="ko-KR">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>헤더 컴포넌트 프리뷰</title>
  </head>
  <body>
    <!-- ⚠️ 프리뷰 전용 페이지입니다. 실제 헤더는 header.js를 사용하세요. -->
    <div id="header-container"></div>

    <script type="module">
      import { renderHeader } from "./header.js";
      renderHeader("#header-container");
    </script>
  </body>
</html>
```

이렇게 하면 **헤더 HTML이 header.js 한 곳에서만 관리**돼요 (높은 응집도!).

> **쉽게 말하면:** 같은 공지사항을 2개의 게시판에 따로 붙이면, 하나를 수정할 때 다른 하나를 잊어버려요. "원본 게시판(header.js)"에만 붙이고, 다른 곳에서는 "원본을 보세요"라고 안내하는 게 좋아요.

## 3-7. package.json에 같은 패키지가 중복으로 있어요 (중요도: 중간)

### 문제

`package.json`에서 `@tailwindcss/vite`와 `tailwindcss`가 `dependencies`와 `devDependencies` 양쪽에 모두 있어요.

### 해결 방법

**Step 1.** 터미널에서 아래 명령을 실행하세요.

```bash
# 먼저 dependencies에서 제거
npm uninstall @tailwindcss/vite tailwindcss

# devDependencies에만 설치
npm install -D @tailwindcss/vite tailwindcss
```

**Step 2.** `package.json`이 아래처럼 되었는지 확인하세요.

```json
{
  "type": "module",
  "devDependencies": {
    "@tailwindcss/vite": "^4.2.2",
    "eslint": "^10.1.0",
    "jest": "^30.3.0",
    "tailwindcss": "^4.2.2",
    "vite": "^8.0.3"
  },
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview",
    "test": "jest",
    "clean": "rm -rf dist build"
  }
}
```

**참고:** `scripts`의 `tailwind`와 `watch` 명령도 제거했어요. Tailwind v4에서는 Vite 플러그인이 자동으로 CSS를 처리하기 때문에, `tailwindcss -i ... -o ...` 같은 CLI 명령이 필요 없어요. `npm run dev` 하나로 전부 처리돼요!

> **쉽게 말하면:** 같은 도구를 두 상자에 넣으면 혼란스러워요. 빌드 도구는 "개발 도구 상자(devDependencies)"에만 넣으면 돼요.

```css
/* 수정 전 (main_list.css 139~141번째 줄) */
.movie-card__star {
  color: #facc15;
}

/* 수정 후 */
.movie-card__star {
  color: var(--color-star);
}
```

**Step 2.** `theme.css`에 아직 없는 색상이 있다면 추가하세요. 현재 `#a3a3a3`과 `#404040`이 변수에 없을 수 있어요.

```css
/* src/styles/theme.css에 추가 */
:root {
  /* 기존 변수들... */
  --background-base: #171717;
  --background-surface: #262626;
  --background-elevated: #404040;  /* ← 추가: 더 밝은 배경 */

  --text-primary: #ffffff;
  --text-secondary: #737373;
  --text-tertiary: #d4d4d4;
  --text-detail: #e5e5e5;
  /* ... */
}
```

**Step 3.** `main_list.css` 맨 위에 `theme.css`를 import해야 CSS 변수를 사용할 수 있어요.

```css
/* main_list.css 맨 위에 추가 */
@import "tailwindcss";
@import "../../styles/theme.css";
```

> **쉽게 말하면:** 색상표(theme.css)를 만들어놓고 "빨간색은 #dc2626이야"라고 정해놨으면, 그림을 그릴 때 "빨간색으로 칠해"라고만 하면 돼요. 나중에 빨간색을 주황색으로 바꾸고 싶으면 색상표만 수정하면 모든 그림이 한 번에 바뀌어요!

## 3-9. Tailwind CSS import 방식 통일 (중요도: 낮음)

### 문제

파일마다 Tailwind를 다른 방식으로 불러오고 있어요. `@import`는 Tailwind의 전체 CSS를 가져오고, `@reference`는 클래스 이름만 참조해요.

### 해결 방법

**Step 1.** 팀에서 규칙을 하나 정하세요.

**추천 규칙:**
- 루트 CSS(`index.css` 또는 `theme.css`)에서 `@import "tailwindcss"` **한 번만** 선언
- 나머지 CSS 파일에서는 Tailwind 클래스를 CSS에서 쓸 때(`@apply` 등) `@reference "tailwindcss"` 사용
- HTML에서 Tailwind 클래스를 직접 쓸 때는 해당 페이지의 CSS에 별도 import 불필요 (루트 CSS에서 이미 로드됨)

```css
/* main_list.css — @import 대신 @reference 사용 (또는 아예 제거) */
/* 수정 전 */
@import "tailwindcss";

/* 수정 후 — Tailwind 유틸리티를 CSS에서 안 쓰면 제거 */
/* (header.css도 마찬가지) */
```

> **쉽게 말하면:** 교과서를 가방에 5권 같은 책을 넣으면 무겁잖아요? 한 권만 넣고 모든 수업에서 공유하면 돼요!

## 3-10. input.js에 `type="module"` 추가 (중요도: 낮음)

### 문제

`src/components/input.html`의 133번째 줄에서 `<script>` 태그에 `type="module"`이 없어요.

### 해결 방법

**Step 1.** `src/components/input.html`의 스크립트 태그를 수정하세요.

```html
<!-- 수정 전 (src/components/input.html 133번째 줄) -->
<script src="./input.js"></script>

<!-- 수정 후 -->
<script type="module" src="./input.js"></script>
```

> **쉽게 말하면:** `type="module"`은 "이 코드를 안전한 개인 방에서 실행해줘"라는 뜻이에요. 없으면 변수들이 밖으로 새어나가서 다른 코드와 충돌할 수 있어요.
