## 3-1. 모든 src 파일이 비어 있어요 (중요도: 높음)

### 문제

`src/` 아래 41개 파일(HTML 14개, CSS 14개, JS 13개)이 전부 비어있어요. 빌드하면 빈 페이지가 생성되고, 개발을 시작할 수 없는 상태예요.

### 해결 방법

**Step 1.** 먼저 랜딩 페이지 HTML에 기본 구조를 넣어주세요.

```html

    <meta charset="UTF-8" />

    <title>Bui1d-BOX - 랜딩</title>
    <link rel="stylesheet" href="./landing.css" />
  </head>
  <body>
    <header>
      <h1>Bui1d-BOX</h1>
      <nav>
        <a href="/src/account/login/login.html">로그인</a>
        <a href="/src/account/signup/signup.html">회원가입</a>
      </nav>
    </header>

    <main>
      <h2>환영합니다!</h2>
      <p>Bui1d-BOX에 오신 것을 환영합니다.</p>
```


## 3-2. index.html이 참조하는 파일들이 존재하지 않아요 (중요도: 높음)

### 문제

`index.html`의 8~15번째 줄에서 favicon, font, 공통 CSS 파일을 불러오려 하지만, 해당 파일/폴더가 프로젝트에 없어요.

### 해결 방법

**Step 1.** `public/` 폴더를 만들어주세요. Vite에서 정적 파일(이미지, 폰트, 아이콘 등)은 `public/` 폴더에 넣어야 해요.

```bash
# 터미널에서 실행
mkdir -p public/favicon
mkdir -p public/font/suit
```

**Step 2.** favicon 파일을 `public/favicon/` 폴더에 넣어주세요.

만약 아직 favicon 파일이 없다면, 임시로 `index.html`에서 해당 줄을 주석 처리하거나 삭제해요:

```html
<!-- 수정 전 (index.html 8~9번째 줄) -->
<link rel="shortcut icon" href="/favicon/likelion.ico" type="image/x-icon" />
<link rel="icon" href="/favicon/likelion.svg" type="image/svg+xml" />

<!-- 수정 후 — 파일이 없으면 주석 처리 -->
<!-- <link rel="shortcut icon" href="/favicon/likelion.ico" type="image/x-icon" /> -->
<!-- <link rel="icon" href="/favicon/likelion.svg" type="image/svg+xml" /> -->
```

**Step 3.** SUIT 폰트 파일을 다운로드하여 `public/font/suit/`에 넣어주세요.

SUIT 폰트는 여기서 받을 수 있어요: https://sunn.us/suit/

다운로드 후:
```
public/
└── font/
    └── suit/
        ├── SUIT-Variable.woff2    ← 폰트 파일
        └── SUIT-Variable.css      ← 폰트 CSS
```

**Step 4.** 공통 CSS 파일을 만들어주세요.

```bash
mkdir -p src/common
```

```css
/* src/common/index.css (새로 생성) */

/* 기본 리셋 */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

/* 공통 스타일 */
body {
  font-family: "SUIT Variable", sans-serif;
  line-height: 1.6;
  color: #333;
}

a {
  text-decoration: none;
  color: inherit;
}
```

> **쉽게 말하면:** HTML에서 "이 파일 가져와!"라고 적어놨는데 실제 파일이 없으면, 택배를 시켰는데 물건이 안 온 것과 같아요. 파일을 만들어주거나, 아직 없으면 적어둔 걸 지워야 에러가 안 나요.

## 3-3. ESLint 설정에 React 플러그인이 들어있어요 (중요도: 중간)

### 문제

`eslint.config.js`의 4~5번째 줄, 20~22번째 줄에서 React 플러그인을 사용하고 있지만, 이 프로젝트는 바닐라 JavaScript 프로젝트예요.

### 해결 방법

**Step 1.** `eslint.config.js` 파일 전체를 아래 내용으로 교체해주세요.

```js
// 수정 전 (eslint.config.js 전체)
import js from "@eslint/js";
import globals from "globals";
import react from "eslint-plugin-react";
import reactHooks from "eslint-plugin-react-hooks";

export default [
  js.configs.recommended,
  {
    files: ["**/*.js", "**/*.jsx"],
    languageOptions: {
      ecmaVersion: "latest",
      sourceType: "module",
      globals: {
        ...globals.browser,
        ...globals.node,
      },
    },
    plugins: {
      react,
      "react-hooks": reactHooks,
    },
    rules: {
      "no-unused-vars": "warn",
      "no-console": "warn",
      "no-debugger": "error",
      eqeqeq: "error",
      curly: "error",
      "prefer-const": "error",
      "react/react-in-jsx-scope": "off",
      "react/jsx-uses-react": "off",
      "react/prop-types": "off",
      "react-hooks/rules-of-hooks": "error",
      "react-hooks/exhaustive-deps": "warn",
      "no-var": "error",
    },
    settings: {
      react: {
        version: "detect",
      },
    },
  },
];

// 수정 후 (eslint.config.js 전체)
import js from "@eslint/js";
import globals from "globals";

export default [
  js.configs.recommended,

  {
    files: ["**/*.js"],
    languageOptions: {
      ecmaVersion: "latest",
      sourceType: "module",
      globals: {
        ...globals.browser,
      },
    },
    rules: {
      /* 기본 코드 품질 */
      "no-unused-vars": "warn",
      "no-console": "warn",
      "no-debugger": "error",

      /* JS 스타일 */
      eqeqeq: "error",
      curly: "error",
      "prefer-const": "error",

      /* 기타 */
      "no-var": "error",
    },
  },
];
```

**Step 2.** 사용하지 않는 React 관련 패키지를 설치하지 않았는지 확인해주세요.

```bash
# 터미널에서 실행 — 이 패키지들이 있으면 제거
npm uninstall eslint-plugin-react eslint-plugin-react-hooks
```

> **쉽게 말하면:** 자전거를 타는데 자동차용 내비게이션을 달아놓은 것과 비슷해요. 동작은 할 수 있지만 불필요한 기능이 끼어들어서 혼란을 줄 수 있어요. 프로젝트에 맞는 도구만 쓰는 게 좋아요!

## 3-4. index.html의 인라인 스타일 (중요도: 낮음)

### 문제

`index.html` 17번째 줄에서 `<body style="margin: 50px">`로 인라인 스타일을 사용하고 있어요.

### 해결 방법

**Step 1.** `index.html`에서 인라인 스타일을 제거하세요.

```html
<!-- 수정 전 (index.html 17번째 줄) -->
<body style="margin: 50px">

<!-- 수정 후 -->
<body>
```

**Step 2.** `src/common/index.css`에 해당 스타일을 추가하세요.

```css
/* src/common/index.css에 추가 */
body {
  margin: 50px;
}
```

> **쉽게 말하면:** 옷에 가격표를 달고 다니는 것과 비슷해요. 스타일 정보는 CSS 파일이라는 "옷장"에 정리해두는 게 깔끔해요!

## 3-5. .gitignore에 추가하면 좋은 항목들 (중요도: 중간)

### 문제

`.gitignore`에 3개 항목만 있어서, `.env` 같은 민감한 파일이나 OS 생성 파일이 실수로 커밋될 수 있어요.

### 해결 방법

**Step 1.** `.gitignore` 파일을 아래 내용으로 교체해주세요.

```gitignore
# 수정 전 (.gitignore)
node_modules/
dist/
build/

# 수정 후 (.gitignore)
# 의존성
node_modules/

# 빌드 결과물
dist/
build/

# 환경 변수 (API 키 등 민감 정보)
.env
.env.local
.env.production

# 에디터 설정
.vscode/
.idea/
*.swp
*.swo

# OS 파일
.DS_Store
Thumbs.db
```

> **쉽게 말하면:** `.gitignore`는 "이 파일은 Git에 올리지 마세요"라는 목록이에요. 비밀번호가 적힌 메모(.env)나 컴퓨터가 자동으로 만드는 쓰레기 파일(.DS_Store)은 올리면 안 되니까 여기에 적어두는 거예요!

## 3-6. PR 템플릿의 체크박스 문법 오류 (중요도: 낮음)

### 문제

`.github/PULL_REQUEST_TEMPLATE.md`의 16~27번째 줄에서 체크박스 문법이 `[]`로 되어있어 GitHub에서 체크박스가 표시되지 않아요.

### 해결 방법

**Step 1.** 모든 `[]`를 `[ ]`로 바꿔주세요 (대괄호 안에 공백 한 칸).

```markdown
<!-- 수정 전 (.github/PULL_REQUEST_TEMPLATE.md 16~27번째 줄) -->
- [] 새로운 기능 추가
- [] 버그 수정

- [] 코드 리팩토링
- [] 주석 추가 및 수정
- [] 문서 수정
- [] 테스트 추가, 테스트 리팩토링
- [] 빌드 부분 혹은 패키지 매니저 수정
- [] 파일 혹은 폴더명 수정

- [ ] 코드 리팩토링
```

## 3-7. package.json에 프로젝트 이름과 설명이 없어요 (중요도: 낮음)

### 문제

`package.json`에 `name`, `version`, `description` 필드가 없어요.

### 해결 방법

**Step 1.** `package.json` 맨 위에 프로젝트 정보를 추가해주세요.

```json
// 수정 전 (package.json)
{
  "type": "module",
  "devDependencies": {
    ...
  }
}

// 수정 후
{
  "name": "bui1d-up",
  "version": "0.1.0",
  "description": "바닐라 JS + Vite 프론트엔드 프로젝트",
  "type": "module",
  "devDependencies": {
    ...
  }
}
```

> **쉽게 말하면:** `package.json`은 프로젝트의 "신분증" 같은 거예요. 이름과 버전이 있어야 "이 프로젝트가 뭔지" 바로 알 수 있어요!

## 3-8. 각 페이지에 JavaScript 파일이 없는 페이지가 있어요 (중요도: 중간)

### 문제

`src/landing/` 폴더만 `landing.js`가 있고, 나머지 8개 페이지 폴더에는 JS 파일이 없어요. API를 호출하려면 각 페이지에 JS 파일이 필요해요.

### 해결 방법

**Step 1.** 아래 파일들을 새로 만들어주세요 (빈 파일이라도 좋아요).

```bash
# 터미널에서 실행
touch src/account/login/login.js
touch src/account/signup/signup.js
touch src/main/main_list/main_list.js
touch src/main/detail/detail.js
touch src/main/genre_more/genre_more.js
touch src/mypage/mypage.js
touch src/paragraph/upload/upload.js
touch src/paragraph/edit/edit.js
```

**Step 2.** 나중에 각 HTML 파일에서 해당 JS 파일을 연결해주세요.

```html
<!-- 각 HTML 파일의 </body> 바로 앞에 추가 -->
<script type="module" src="./login.js"></script>
```

`type="module"`을 쓰면 `import`/`export` 문법을 사용할 수 있어요. 그래야 `src/API/` 폴더에 만들어둔 API 코드를 불러올 수 있어요!

> **쉽게 말하면:** HTML은 "뼈대", CSS는 "옷", JS는 "두뇌"에요. 두뇌(JS)가 없으면 버튼을 눌러도 아무 일도 일어나지 않아요. API를 호출하는 건 두뇌의 역할이에요!
